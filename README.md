> # Assignment Questions
>
> ## Assignment 1: Multi-Vendor Electronics E-Commerce Platform
>
> ## Objective
>
> Develop a complete full-stack MERN application for a multi-vendor electronics marketplace where vendors can manage and sell products while customers can browse, purchase, and track orders. The application should demonstrate frontend development, backend API creation, database management, authentication, and deployment skills.
>
> ## Tech Stack to be Used
>
> Frontend
>
> React.js
>
> React Router
>
> Redux Toolkit or Context API
>
> Axios
>
> CSS / Tailwind CSS / Material UI
>
> Backend
>
> Node.js
>
> Express.js
>
> Database
>
> MongoDB with Mongoose
>
> Authentication & Security
>
> JWT Authentication
>
> bcrypt for password hashing
>
> Deployment
>
> Vercel / Netlify for Frontend
>
> Render / Railway / Cyclic for Backend
>
> MongoDB Atlas for Database
>
> ## Functionalities to Implement
>
> 1. Authentication and Role Management
> Implement secure JWT-based authentication with separate roles for Admin, Vendor, and Customer. Protected routes should restrict access based on user roles and permissions.
> 2. Vendor Product Management
> Vendors should be able to create, edit, update, and delete products. Product details must include title, description, category, price, stock quantity, and product images.
> 3. Product Search and Filtering
> Customers should be able to search products using keywords and apply filters such as category, price range, availability, and ratings for easier navigation.
> 4. Shopping Cart Management
> Users should be able to add products to the cart, update quantities, remove items, and view total pricing dynamically.
> 5. Wishlist Functionality
> Customers should be able to save products to a wishlist for future purchases without adding them directly to the cart.
> 6. Order Placement and Tracking
> Implement complete order flow from checkout to order confirmation. Customers should be able to track order status such as Pending, Confirmed, Shipped, and Delivered.
> 7. Payment Workflow
> Create a mock payment integration flow where users can place orders and receive payment confirmation with transaction status handling.
> 8. Review and Rating System
> Customers should be able to provide ratings and reviews for purchased products. Reviews should be displayed on the product details page.
> 9. Dashboard Analytics
> Admin and Vendors should have dashboards displaying statistics such as total products, total orders, revenue summary, and recent activity.
> 10. Responsive Frontend UI
> The frontend should be fully responsive and optimized for desktop, tablet, and mobile devices with proper navigation and reusable components.
>
> ## Key Takeaways
>
> Understanding of complete MERN architecture
>
> Experience in building secure REST APIs
>
> Knowledge of role-based authentication and authorization
>
> Hands-on experience with MongoDB schema relationships
>
> Frontend and backend integration using APIs
>
> Deployment and production-level configuration
>
> ## Important Notes
>
> The project must be developed completely using the MERN stack
>
> Do not use Firebase or Supabase as backend replacements
>
> Proper folder structure and clean coding practices are mandatory
>
> All APIs should include validation and proper error handling
>
> Environment variables must be used for sensitive credentials
>
> Responsive design is compulsory
>

>

>

>
> Screen recording or demo video explaining the application workflow
